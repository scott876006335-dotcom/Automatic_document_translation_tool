# 文件自动翻译工具 — 使用说明

本工具在本地通过 **LM Studio**（OpenAI 兼容 API）或 **Ollama** 调用大模型，对 Word、Excel、PowerPoint、PDF 等文档中的文本进行批量翻译，并生成带编号对齐的新文件。

---

## 一、运行前准备

1. **安装依赖**（在项目目录下）  
   ```text
   pip install -r requirements.txt
   ```
   若项目中无该文件，请至少安装：`python-docx`、`python-pptx`、`openpyxl`、`PyMuPDF`（fitz）等（以程序实际报错为准）。

2. **启动推理服务（二选一）**  
   - **LM Studio**：加载模型后，在「本地服务器」中开启 API（常见地址 `http://127.0.0.1:1234`）。  
   - **Ollama**：安装并启动服务，确保命令行 `ollama list` 可用。

3. **启动程序**  
   运行主入口（例如 `python gui/main_window.py` 或您打包后的可执行文件），出现图形界面后即可使用。

---

## 二、全局参数设置

| 项目 | 说明 |
|------|------|
| **推理后端** | 选择 LM Studio 或 Ollama。选 LM Studio 时需填写服务地址（一般无需带 `/v1` 后缀，程序会自动处理）。 |
| **模型名称** | 与 LM Studio / Ollama 中加载的模型名**完全一致**。 |
| **源语言 / 目标语言** | 下拉选择；翻译提示词会按该语言对生成。 |
| **输出版式** | **原文与译文对照**：在文档中保留原文并追加/并列译文（依格式而定）。**仅译文**：用译文替换原文。 |
| **翻译行业** | 选择文档所属领域，用于在提示词中加入**领域说明**，使术语与风格更贴合（如轨道交通信号、建筑、通用等）。 |
| **自定义领域** | 仅当行业选择为 **「自定义」** 时可编辑。请用简短中文或中英文描述领域（例如：`港口起重机液压系统运维手册`）。不填则按「通用」处理。 |
| **专业词汇表** | 可选：选择一个 **`.txt` 词汇表文件**路径；有有效内容时，会一并写入提示词（见下文第三节）。 |
| **每批段落数** | 单次请求合并送入模型的段落条数（默认 **20**，可在 **4～48** 之间调整）。**不要设得过大**：一批内原文 + 提示词 + 模型输出会占用大量上下文，过大容易触发 **token 上限截断**、漏译或格式错乱；若文档单段很长，宜适当**减小**该值。 |
| **思考模式** | 是否在 API 中请求模型的「思考」能力。**当前仅建议在 LM Studio 下配合 Qwen3.5 系列**使用；其他后端或模型可能忽略该参数或报错后自动回退。详见下文「配置文件」与常见问题。 |

开始翻译时，当前的语言、版式、行业、自定义领域、词汇表路径、**每批段落数**与 **思考模式** 会写入 `config/config.json`，下次启动会恢复。

## 三、个人专业词汇表（.txt）

用于指定「原文 → 译文」的固定对译，在模型翻译时**优先参考**（仍以句意为准，明显冲突时模型可自然翻译）。

### 3.1 应使用的书写格式（务必按此写）

| 项目 | 要求 |
|------|------|
| **文件类型** | 纯文本 **`.txt`**，一行一条词条，不要用 Excel/Word 另存为「假 txt」。 |
| **编码** | 优先保存为 **UTF-8**（推荐无 BOM；带 BOM 也可）。程序解码失败时会再尝试 GBK。 |
| **单行结构** | **`原文:译文`** —— 中间使用**英文半角冒号** `:`（不是中文 `：`）。 |
| **如何分左右** | 每行按 **第一个** `:` 分成左右两段：左边整体算「原文」，右边整体算「译文」。若译文里必须出现冒号，只影响**第一个** `:` 左侧为原文、右侧可含多个 `:`。 |
| **空格** | 左右两侧**首尾空格**会被自动去掉；中间空格保留。 |
| **注释** | 行首（去掉首尾空格后）以 **`#`** 开头的整行视为注释，**不参与**解析。 |
| **空行** | 忽略。 |

**标准示例（可复制改）：**

```text
# 专业词汇表 — 每行：原文:译文

CBTC:基于通信的列车控制
interlocking:联锁
ATP:列车自动防护
Note: this is OK: 右侧可含英文冒号
```

上例最后一行解析为：原文 `Note`，译文 `this is OK: 右侧可含英文冒号`（因只认第一个 `:`）。

### 3.2 注意事项（容易踩坑）

1. **语言要与界面一致**  
   「原文」应对应界面 **源语言**，「译文」应对应 **目标语言**。例如源语言选英语、目标语言选简体中文，则左侧写英文术语、右侧写中文定译；若反了，模型很难按预期统一。

2. **不要用中文全角冒号 `：` 当分隔符**  
   程序只认英文 `:`。若写成 `CBTC：基于通信…`，整行可能因解析规则被忽略或切分错误（取决于是否含半角 `:`）。

3. **不要一行塞多条**  
   每条独占一行；不要用分号、顿号把多对词连在一起，程序不会按那种规则拆分。

4. **左右不能为空**  
   `:` 左侧或右侧去掉空格后若为空，**整行丢弃**。例如 `:译文` 或 `原文:` 都无效。

5. **无冒号的行无效**  
   整行没有 `:` 会被忽略（注释行除外）。

6. **词条宜短、宜专**  
   优先写**术语、缩写、固定产品名**；不要整段句子。句子级「原文」在正文里很难整段匹配，效果差。

7. **大小写与变形**  
   模型会按「接近匹配」理解词条；英文大小写不完全一致时仍可能生效，但不保证。重要术语建议**与文档里常见写法一致**。

8. **生效条件**  
   须在界面选好词汇表路径，且文件存在、且至少有一行有效词条，才会把词汇表送进模型；否则**完全不使用**词汇表。

9. **长度上限**  
   单次翻译会限制词条条数与总字符数（约数百条、约万余字符量级），**靠后的词条可能被截断**不进入当次提示。词条很多时请拆成多个文件分次使用，或只保留最关键的词。

10. **与「翻译行业」的关系**  
    词汇表是**硬性对译提示**；「翻译行业」是**领域风格**。两者可同时使用，互不替代。

### 3.3 何时不会使用词汇表

- 未选择 `.txt` 路径，或路径指向的文件不存在。  
- 文件为空，或全部是空行/注释/无效行。  
- 上述任一情况：**不会**向模型附加词汇表段落，翻译仍正常进行。

### 3.4 使用建议

- 只收录**真正需要统一译名**的术语，不必把整个词典贴进去。  
- 定期核对文档里实际出现的写法，词条与正文**用词一致**时效果最好。

## 四、单文件模式

1. 点击 **浏览** 选择单个文件（支持常见 Office 与 PDF 等，以程序提示为准）。  
2. 点击 **开始翻译**。  
3. 完成后会弹出提示；输出文件默认保存在**同目录**，文件名由多段用下划线 `_` 连接，**最后一段固定为 `XuYuan`**（作者署名），形如：  
   `原文件名_源语言代码_目标语言代码_翻译行业_版式_[词汇表主文件名]_XuYuan.扩展名`  
   - **翻译行业**：与界面「翻译行业」下拉选项一致的中文名称（如 `通用`、`轨道交通信号`）；「自定义」时仍显示为「自定义」，自定义说明不写入文件名。  
   - **版式**：`双语对照` 或 `纯译文`（与界面「输出版式」一致）。  
   - **词汇表主文件名**：仅当选择了词汇表文件、且文件中至少有一对有效 `原文:译文` 时才会插入（不含 `.txt` 扩展名，非法字符会替换为 `_`，长度会截断）。未使用或无有效词条时**省略该段**。  
   批量处理时会**跳过**文件名中包含 `_XuYuan` 的文件，避免重复翻译输出。

---

## 五、批量目录模式

1. 选择**目标文件夹**。  
2. 勾选要处理的 **文件类型**（Word / Excel / PPT / PDF）。  
3. 点击 **批量翻译**。  
4. 子目录会递归扫描；临时文件（如 `~$` 开头）及已生成的 `*_…_XuYuan.*` 会跳过。

## 七、配置文件

- 路径：`config/config.json`。  
- 可手动编辑 **LM Studio 超时**、**重试次数**、**默认模型**、**`translation.batch_size`（每批段落数）**、**`lm_studio.enable_thinking`（思考模式）** 等（与 `utils/config.py` 中默认结构对应）。  
- **`translation.batch_size`**：与界面「每批段落数」一致；**不宜过大**，否则单批输入输出易超出服务端 **max_tokens / 上下文** 限制，出现截断或解析失败。  
- **`lm_studio.enable_thinking` / `lm_studio.send_chat_template_kwargs`**：思考模式依赖向 LM Studio 传入 `chat_template_kwargs`（如 `enable_thinking`）。**目前仅在实际验证过的组合下推荐使用：LM Studio 本地服务 + Qwen3.5 系列模型**；若服务端不支持该字段，程序可能按原有逻辑重试或省略参数。  
- 修改 JSON 时注意逗号与引号，保存为 UTF-8。

## 八、常见问题

1. **连接 LM Studio 失败**  
   确认软件已启动、本地服务器已开启、端口与界面填写一致。

2. **Ollama 连接失败**  
   确认 Ollama 服务已运行，且本机可执行 `ollama list`。

3. **翻译质量不理想**  
   尝试更换更强模型；调整 **翻译行业** 或 **自定义领域**；补充 **专业词汇表**；检查源/目标语言是否选反。

4. **词汇表未生效**  
   检查路径是否正确、文件是否为 UTF-8、每行是否为 `原文:译文` 且两侧非空。

5. **每批段落数与 token / 截断**  
   若出现译文被截断、批次重试增多或 LM Studio 报错与长度相关，请**减小**「每批段落数」，并检查 `translation.max_tokens_per_batch` 与服务端限制是否匹配。

6. **思考模式无效果或报错**  
   请确认使用 **LM Studio** 且加载 **Qwen3.5** 系列；其他环境可能不支持。若 HTTP 400 等与 `chat_template_kwargs` 相关，可在 `config.json` 中调整 `lm_studio.send_chat_template_kwargs`（详见 `utils/config.py` 默认值），或关闭「思考模式」。
